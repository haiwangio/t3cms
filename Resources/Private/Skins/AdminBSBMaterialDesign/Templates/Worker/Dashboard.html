Why we count on FLUX!

Content is strict seperated from the websites layout and design. That is a good way to create powerful TYPO3 themes.
So the content elements are not realy part of the TYPO3 theme, because you can always create HTML elements and fit the content elements HTML code in it. So if you want, you can create extra extensions to deliver your content elements. We were thinking about to use EXT:mask, but the workflow with mask was not effizient enougth. The EXT:mask_export made mask to a better option, but with EXT:mask_export we loose the flexibility for changing/adding/removing fields. The EXT:dce is also not that export friendly for content elements. With EXT:flux elements, we just need one HTML file to create a complex content element. So the workflow is effizient enougth and the flexibility for fields is given, because all configuration is done in this one HTML file. The last big reason is, that FLUX can create Content + IRRE + Grid-Elements

Your backend module can look like this!

Just use layoutRootPaths.10 to change the layoutRootPath in your Extension and use the T3CMS Backend Module Layout:

# TypoScript SETUP
module.tx_yourcoolextension {
    view {
        layoutRootPaths.0 = EXT:your_cool_extension/Resources/Private/Layouts/
        layoutRootPaths.10 = EXT:t3cms/Resources/Private/Skins/AdminBSBMaterialDesign/Layouts/
    }
}

And use the BeDefault layout in your html templates:

<f:layout name="BeDefault" />

Check out the static demo to explore, what elements are available with this be module layout. Open static demo. Note: The t3cms backend module layout is a very slim fork of the original template.

Themes

TYPO3 Themes sind Extensions/Sitepackages, die mit "t3themes_" beginnen, und die "t3themes_conf" Theme Einstellungen verwenden.

Ein TYPO3-Theme benutzt die Theme Einstellungen für das Frontend Layouting, Backend-Layouts werden nur für das Einblenden von Content-Spalten im Backend gebraucht. Um im Frontend Sidebars ein-/aus zu blenden, werden die Theme Einstellungen benutzt.

Themes are using this typoscript snippet to get the themeConf variable into their fluidtemplates:

### DATA PREPROCESSING
dataProcessing {
    ...
    30 = SalvatoreEckel\T3cms\DataProcessing\T3themesConfProcessor
    30 {
        fieldName = t3themes_conf
        as = t3themesConf
    }
}

Learn more about your theme by checking out the themes help module.
